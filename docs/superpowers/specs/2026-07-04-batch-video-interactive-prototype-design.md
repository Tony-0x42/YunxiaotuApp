# 批量剪辑 可交互 HTML 原型设计文档

## 1. 目标与范围

### 1.1 目标
基于 `spec/modules/app` 与 `spec/modules/web` 下的全部 README 需求文档，制作一个可在浏览器中直接打开并交互的 HTML 原型，用于向客户演示批量剪辑 APP 端与管理后台的核心页面流程。

### 1.2 覆盖范围
- **APP 端**：登录、注册、首页、学习专区、品牌专区、信息咨询、新手指南、AI 云创（素材库、视频组编辑、分割进度）、AI 去水印、我的、账号资料编辑、联系客服、用户协议、隐私政策。
- **Web 管理后台**：登录、客户管理（列表、新增账号、账号详情、注册二维码）、会员VIP管理、首页内容管理、公告管理、教程管理、文档管理、数据统计、系统配置。

### 1.3 不覆盖范围
- 真实后端接口对接。
- 复杂视频剪辑/分割算法演示。
- 用户权限鉴权的真实实现。

## 2. 方案选择

### 方案 A：单文件 HTML + Tailwind CSS + 原生 JS（推荐）
- **优点**：仅一个 `.html` 文件，客户无需部署，双击即用；所有页面通过 JS 动态切换；便于邮件/微信转发；维护简单。
- **缺点**：文件体积较大；不适合长期演进为真实项目。
- **适用性**：原型演示场景最佳。

### 方案 B：多页面静态 HTML
- **优点**：更接近真实网站结构，URL 可区分页面。
- **缺点**：文件数量多，分享和同步繁琐，演示时容易遗漏页面。

### 方案 C：Vue/React 轻量脚手架
- **优点**：组件化、可扩展。
- **缺点**：需要构建步骤和依赖安装，客户演示成本高，偏离"原型"定位。

**决策**：采用方案 A，输出单一 HTML 文件，内置所有页面、示例数据和交互逻辑。

## 3. 原型结构与交互说明

### 3.1 APP 端布局
- 整体页面中央展示一个模拟手机屏幕（375×812px 或响应式）。
- 一级页面底部固定 4 项导航：首页 / AI 云创 / AI 去水印 / 我的。
- 二级页面顶部显示返回按钮 + 页面标题。
- 当前选中 tab/菜单高亮为蓝色。

### 3.2 Web 管理后台布局
- 顶部固定导航栏：Logo + 管理员信息 + 退出。
- 左侧固定侧边栏：客户管理、会员VIP管理、首页内容管理、公告管理、教程管理、文档管理、数据统计、系统配置。
- 右侧内容区展示对应模块页面。
- 列表页包含搜索框、筛选条件、新增按钮、数据表格、分页器。

### 3.3 页面跳转关系
- 从 Web 后台登录页点击登录后进入「数据统计」作为后台首页。
- 从 APP 登录页点击登录后进入 APP 首页。
- APP 底部导航切换一级页面。
- 各功能入口点击后进入对应二级页面，二级页面返回上一级。

### 3.4 关键交互模拟
- **登录/注册**：表单校验提示、登录成功后跳转。
- **轮播图**：首页顶部轮播支持自动切换和指示器。
- **AI 去水印**：输入链接后模拟解析过程，展示结果页签切换。
- **AI 云创**：视频组列表、搜索、新增、进入编辑页、模拟 AI 生成按钮状态。
- **Web 客户管理**：表格数据、新增账号弹窗/页面、账号详情 Tab 切换。
- **Web 数据统计**：数据概览卡片 + 图表占位 + 明细表格 + Tab 切换。
- **Web 系统配置**：Tab 切换品牌配置/版本管理/全局参数。
- **弹窗**：高危操作二次确认、成功提示。

## 4. 视觉规范

- **主色调**：蓝色（#3B82F6 / #2563EB），所有主操作按钮蓝色填充。
- **辅助色**：灰色（#6B7280、#9CA3AF）、红色（删除/禁用）、绿色（成功/启用）。
- **字体**：系统默认无衬线字体，中文优先使用系统字体。
- **圆角**：输入框、按钮、卡片统一圆角（4~12px）。
- **阴影**：卡片使用轻微阴影（shadow-sm / shadow-md）。

## 5. 示例数据

原型中内置模拟数据，包括：
- 登录页 Logo/Slogan。
- 首页轮播图、喜报、功能入口、新手指南。
- AI 云创视频组列表与算力数据。
- AI 去水印解析结果示例。
- 我的页面账号信息、VIP 信息、客服信息。
- Web 后台客户管理账号列表、VIP 列表、公告/教程/文档列表、统计数据等。

## 6. 技术选型

- **HTML5**：单一入口文件。
- **Tailwind CSS v4 CDN**：快速样式开发。
- **原生 JavaScript**：页面切换、数据渲染、简单交互。
- **无外部构建工具**：无需 npm/webpack，客户可直接打开。
